# Release notes 1.x.x

The standard Server Pro license allows you to run the application in a production environment as well as one in a non-production/sandbox environment; it is highly recommended that you provision a non-production environment for testing.

#### [hashtag](release-notes-1.x.x.md#server-pro-1.2.1) Server Pro 1.2.1

(Community Edition Image ID: `8e1bae81acac`)

#### [hashtag](release-notes-1.x.x.md#server-pro-1.2.0) Server Pro 1.2.0

* Use both 2016 and 2017 versions of texlive, for backward compatibility

#### [hashtag](release-notes-1.x.x.md#server-pro-1.1.0) Server Pro 1.1.0

* Update texlive to 2017.1
* Use `https` for the learn-wiki proxy by default

#### [hashtag](release-notes-1.x.x.md#server-pro-1.0.2) Server Pro 1.0.2

* A new [auto-compilearrow-up-right](https://www.overleaf.com/blog/auto-compiling-in-sharelatex-2017-10-31) option, keep the pdf view in sync while you type
* A new [link-sharingarrow-up-right](https://www.overleaf.com/blog/integration-update-link-sharing-2017-11-27) feature, replacing the public/private permissions system
* New administration interface
* Package-aware Autocomplete
* Various bug fixes and performance improvements

#### [hashtag](release-notes-1.x.x.md#server-pro-1.0.0) Server Pro 1.0.0

* Upgraded all internal services to latest node-js runtime
* Per-User Track Changes
* Improved Autocomplete
* Improved Spellcheck
* Fixed an issue with creating templates
* Enable SES email configuration via [Instance Rolesarrow-up-right](https://github.com/overleaf/overleaf/wiki/Configuring-SMTP-Email#aws-ses-with-instance-roles)
* Various bug-fixes and improvements
